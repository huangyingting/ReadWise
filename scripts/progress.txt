## Codebase Patterns
- Stack: Next.js 15 (App Router, TS), React 19, Prisma+SQLite, NextAuth v4 + @auth/prisma-adapter (DB session strategy).
- Admin enforcement: pages -> `requireAdmin(callbackUrl)` from `@/lib/session` (redirects non-admins to `/forbidden`); API routes -> `requireAdminApi()` from `@/lib/api-auth` (returns `{error: NextResponse}` 401/403, else `{session}`). Hide admin UI via `session.user.role === "Admin"`.
- First-user-becomes-Admin: handled in `events.createUser` in `src/lib/auth.ts` (NOT a callback). Real OAuth sign-in triggers it.
- DB lives at `prisma/dev.db` (DATABASE_URL `file:./dev.db` is resolved relative to the schema dir).
- Run node/prisma scripts from the PROJECT ROOT so `@prisma/client` resolves (not from /tmp).
- Verify role-gated routes without real OAuth: insert a `User` + `Session` (sessionToken) row, then send cookie `next-auth.session-token=<token>` to the live server.
- Import the singleton `prisma` from `@/lib/prisma`; never instantiate PrismaClient directly.
- Auth: `authOptions` in `src/lib/auth.ts`. Providers added conditionally on env vars (missing creds => provider omitted, no crash).
- Session is DATABASE strategy. Attach `id`/`role` in the `session({session,user})` callback. Session type augmented in `src/types/next-auth.d.ts`.
- Protected routes: add prefix to BOTH `middleware.ts` (cookie-presence redirect to `/signin?callbackUrl=`) AND call `requireSession(callbackUrl)` from `@/lib/session` in the server component for real enforcement.
- Next 15 server components: `await searchParams` / `await params` (they are Promises).
- Run prisma/build/dev with env loaded: `set -a && . ./.env.local && set +a`.
- Migrations committed under `prisma/migrations/`; `dev.db` is gitignored.
- Browser verify with Playwright run from project root; chromium `--no-sandbox` at /home/azadmin/.cache/ms-playwright/chromium-1228/chrome-linux64/chrome (the chromium-1208/chrome-linux path in early notes is stale; $HOME=/home/azadmin).
- Always render stored article HTML through `sanitizeArticleHtml` (src/lib/sanitize.ts) before `dangerouslySetInnerHTML`; never inject raw `content`. Article reader is `/reader/[id]`; data helpers in src/lib/articles.ts.
- Reading progress: `ReadingProgress` (userId+articleId @@unique). Persist via `saveProgress` in src/lib/progress.ts — FORWARD-ONLY, sticky `completed` at COMPLETION_THRESHOLD=95. Use `getProgressMap(userId, ids)` for listings and the server `ArticleCard` component to render bars. Scroll tracker is the client `ReaderProgress` (throttled <=1/sec, no auto-scroll).
- AI calls: use `src/lib/ai.ts` `chatComplete(messages, {maxOutputTokens})` (Azure OpenAI over fetch, no SDK) — returns text or `null`; gate features with `isAiConfigured()`. The gpt-5-mini deployment needs `max_completion_tokens` (NOT `max_tokens`), no custom `temperature`. Cache AI outputs in a dedicated table keyed by natural inputs and NEVER cache graceful fallbacks (so real output can replace placeholders later). Run Playwright scripts from the project root so `@playwright/test` resolves.
- AI structured output: prompt for "JSON array only", then before JSON.parse strip ``` fences and slice between the first `[` and last `]` (models sometimes wrap output); dedup parsed entries by lowercased key. See `parseVocabularyJson` in src/lib/vocabulary.ts.
- Per-user vs shared AI caches: cache expensive AI extraction by article (shared, e.g. `VocabularyItem`) and store per-user state separately (e.g. `SavedWord` with `@@unique([userId, word])` for dedup); merge user flags at read time via a `getSavedWordSet`-style helper.
- Azure Speech TTS (word-level timing): use the Speech SDK (`microsoft-cognitiveservices-speech-sdk`), NOT the REST TTS endpoint (which returns audio only). `SpeechSynthesizer(cfg, null)` returns audio in `result.audioData` headless; `synthesizer.wordBoundary` (boundaryType===Word) gives `textOffset`/`wordLength` into the input text + audioOffset/duration in TICKS (/1e7 => seconds). Routes using the SDK need `export const runtime = "nodejs"`. Store audio as base64 + return a `data:` URL (no audio-serving route). See `src/lib/speech.ts`. Headless audio playback in Playwright needs `--autoplay-policy=no-user-gesture-required`.
- Client listing feeds (e.g. `CategoryBrowser`) seed `useState` from server props ONCE; Next soft-nav between views changes props but does NOT reset state — pass `key={activeView}` on the page so the component remounts per view. To render cards in a client loader, use a presentational client component (`ArticleCardView`) fed a plain serializable object via `toListingArticle(article)`; never import prisma-backed helpers into client code. Offset pagination helpers (`listCategoryPage`/`listPicksPage`) use `take: limit+1` to compute `hasMore`.
